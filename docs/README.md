# Documentation Workflow

This project keeps two kinds of docs as we build:

1. `docs/decisions/` for architecture and tradeoff decisions.
2. `docs/how-to/` for practical setup and usage guides.
3. `docs/learn/` for milestone-by-milestone learning notes.

## What to expect from me each milestone

- A short implementation summary.
- Why we chose that approach.
- Key concepts to learn (Docker, Django, Next.js, DB, etc.).
- Commands you can run to validate behavior.
- Common pitfalls and debugging tips.

## Naming convention

- Decisions: `docs/decisions/YYYY-MM-DD-<topic>.md`
- How-to: `docs/how-to/<topic>.md`
- Learning notes: `docs/learn/milestone-<n>-notes.md`
